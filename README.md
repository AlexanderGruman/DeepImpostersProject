# Deep learning based analysis of edgent hebrew sources

This project is a research study rather than a development project.

Our goal is to investigate and try to enlighten new knowledge about the authorship of ancient Jewish texts—using existing computational methods as analytical tools.

Along the way, we will explore the following:
- How to adapt natural language processing tools to the unique challenges of ancient texts
- Determine the appropriate methods for analyzing the material
- And carefully select which historical writings are suitable for comparison

Keywords: Authorship attribution in Jewish texts, Natural Language Processing (NLP), Deep imposters approach, Deep Learning for Stylometry.

## Project Overview
This research centers on writing-style analysis and authorship attribution within ancient Hebrew texts, specifically focusing on the Five Books of the Torah. By extracting subtle stylistic fingerprints, the system transforms raw historical text into continuous numerical sequences (stylistic signals) to detect anomalies, similarities, and structural shifts.

The framework implements two primary advanced architectures:
1. **CNN-Based Pipeline:** Utilizing Convolutional Neural Networks coupled with the *Deep Impostors* approach to capture local stylistic patterns across text blocks.
2. **Sen2Pro Pipeline:** Representing sentences and segments as probabilistic distributions rather than static vectors, making it highly robust against rare words, non-modern semantic structures, and morphological ambiguities inherent to ancient Hebrew.

## Repository Structure & Core Components

This repository contains the complete experimental pipeline and visualization notebooks:

* **`Pipeline Execution Notebook (CNN)`**: Contains the training, evaluation, and tokenization framework for the CNN-based Deep Impostors model.
* **`Pipeline Execution Notebook (Sen2Pro)`**: Implements the probabilistic text representation, distance calculations, and the Sen2Pro analytical pipeline.
* **`Graphics & Visualizations Notebook`**: Generates corresponding charts, comparative plots, and evaluation metrics mapping out the outputs generated by both pipelines.

## 🚨 Critical Setup Instruction: Manual File Download

Due to GitHub's file size limitations, a large pre-trained word embedding vector file required for tokenization and model initialization **is not included in this repository**. 

You **must** download and place this file manually before running any of the execution notebooks.

### Steps to set up:
1. Download the pre-trained FastText Hebrew vector file from Facebook's public file server:
   👉 [cc.he.300.vec.gz](https://dl.fbaipublicfiles.com/fasttext/vectors-crawl/cc.he.300.vec.gz)
2. Place the downloaded `cc.he.300.vec.gz` file directly into the root/designated project directory.
3. Verify that your environment path points correctly to this archive file.
